## 核心理念

> **"Remember well. Connect everything."**
> **记得清晰，连接万物。**

Liora 重新定义记忆的方式。不是简单的日记，不是孤立的关系图谱——而是**活的、互联的、不断生长的记忆网络**。

上传一段文字、一张照片、一段录音，AI 自动解析其中的实体与关系，织成一张属于你的知识图谱。跨越时间、空间、情感的维度，帮你发现记忆之间那些隐秘却动人的联系。

## 功能详解

### 🕸️ 1. 知识图谱可视化

**D3.js 力导向图谱引擎**，将你的记忆网络以动态图形的形式呈现。

**核心能力：**
- **力导向布局** — 节点根据关系自动排列，相似实体自然聚类
- **拖拽交互** — 拖动节点调整位置，布局自动重新平衡
- **缩放与平移** — 滚轮缩放画布，鼠标拖拽平移视角
- **节点搜索** — 输入关键词快速定位实体，图谱自动聚焦
- **详情面板** — 点击任意节点，查看实体描述、关联记忆、关系列表
- **关系编辑** — 支持修改关系类型、添加描述、删除错误关系
- **实体合并** — 将两个重复实体合并为一个，自动去重关系

**图谱节点类型：** 人物（PERSON）、地点（LOCATION）、事件（EVENT）、通用实体（ENTITY）

**适用场景：**
- 梳理人物关系网
- 回顾某一时期的经历
- 发现记忆中的隐藏联系

---

### 📝 2. 多模态记忆录入

**支持三种输入形式**，AI 自动完成内容理解与知识抽取。

**文字录入：**
- 自由文本输入，记录想法、事件、感悟
- 支持 @提及 已有实体，自动建立关联
- 支持 #话题标签，对记忆进行分类

**图片上传：**
- 照片、截图、扫描件均可
- AI 识别图片中的文字（OCR）和实体
- 支持 JPEG / PNG / GIF 等常见格式

**音频录制：**
- 直接在浏览器中录制语音
- 自动语音转文字（依赖 LLM 能力）
- 提取关键信息和情感基调

**AI 自动分析：**
录入内容后，Liora 会自动完成：

| 分析维度 | 说明 |
|:---------|:-----|
| 实体抽取 | 识别人名、地名、机构、事件等实体 |
| 关系抽取 | 解析实体之间的关系（朋友、位于、参加等） |
| 情感分析 | 判断记忆的情感倾向（正面 / 中性 / 负面）与强度 |
| 摘要生成 | 生成记忆的一句话概括 |
| 时间解析 | 自动识别记忆发生的时间（精确到日或月年） |

---

### 🔍 3. 混合语义搜索

**向量检索 + 关键词匹配双引擎**，兼顾语义理解与精确命中。

```
用户查询
    │
    ├── 向量搜索（FAISS）
    │       └── 将查询转为向量，在记忆中找出语义最相似的
    │           权重 70%
    │
    └── 关键词搜索（BM25）
            └── 精确匹配查询中的关键词
                权重 30%
                    │
                    └── 综合评分排序输出
```

**特点：**
- **语义理解** — 搜索"电影"能找到包含"看电影"、"电影院"的记忆
- **多结果融合** — 两种搜索结果取并集，按综合分数排序
- **实体匹配** — 同时搜索图谱节点名称，高亮匹配实体
- **结果标注** — 显示匹配类型（语义匹配 / 关键词匹配 / 两者都有）

---

### 📅 4. 历史上的今天

**穿越时间的记忆回溯**，发现往年今日的珍贵连接。

**功能逻辑：**
1. 系统比对记忆中记录的时间与当前日期（月 + 日）
2. 筛选出往年同一天（不同年份）的记忆
3. 按时间倒序展示，最早的年份排在最前
4. 如果没有往年今日记忆，随机展示其他记忆作为惊喜

**视觉呈现：**
- 复古风格时间戳
- AI 洛忆为每条记忆生成温暖评语
- 支持导出为「记忆卡片」分享

---

### 🃏 5. 复古记忆卡片

**档案美学风格的记忆导出**，每一张都是时光的礼物。

**卡片内容：**
- 记忆摘要（一句话概括）
- 原始内容（节选）
- 时间信息（用复古日历风格呈现）
- AI 洛忆的温暖回应
- 实体标签（以档案标签样式展示）

**导出方式：**
- 使用 html2canvas 将卡片导出为 PNG 图片
- 可保存到本地或分享给他人

**设计风格：**
- 米黄色纸张背景 + 深棕色文字
- 圆形印章样式的日期
- 打字机字体的正文
- 档案编号与分类标签

---

### 🤖 6. AI 洛忆

**你的专属智能伙伴**，懂得你的记忆网络，给出有温度的回应。

洛忆是 Liora 系统中的 AI 伙伴。它不是通用助手，而是专门为你的记忆网络训练的回应引擎。它知道你记录过什么，能将新的记忆与过去的经历串联起来。

**洛忆会根据记忆的情感色调调整自己的语气：**

| 情感基调 | 洛忆风格 | 示例 |
|:---------|:---------|:-----|
| 愉悦（valence > 0.3） | 调侃、搞怪、轻松 | "好家伙，这波装到了！" |
| 悲伤（valence < -0.3） | 温暖、共情，理解 | "抱抱，那时候挺难的吧" |
| 中性 | 温和、陪伴、朋友 | "这种日子挺好的" |

---

### 📊 7. 统计面板

**全局视角的记忆数据分析**，了解你的记忆网络全貌。

| 指标 | 说明 |
|:-----|:-----|
| 记忆总数 | 累计录入的记忆条数 |
| 实体总数 | 图谱中识别的独立实体数 |
| 关系总数 | 实体之间的连接数 |
| 近30天趋势 | 每日记忆录入数量的折线图 |
| 实体类型分布 | 人物 / 地点 / 事件 / 其他的饼图 |
| 情感分布 | 正面 / 中性 / 负面记忆的比例 |
| 最活跃实体 TOP5 | 关联记忆最多的实体排名 |

---

### 🔮 8. 智能预测

**基于图谱结构的智能洞察**，发现记忆之间潜在的关联。

```
当前节点
    │
    ├── 分析该节点的邻居关系类型
    │
    ├── 根据图谱结构模式预测可能的下一节点
    │       （例如：人物A → 参加 → 活动X
    │        预测：人物A 可能还会参加其他活动）
    │
    └── 返回置信度最高的 N 个预测
```

**交互方式：**
- 在图谱中选中任意节点，查看「智能预测」建议
- 每个预测显示：目标实体名、关系类型、预测理由、置信度
- 一键采纳预测，将其正式添加到图谱中

---

### 📤 9. 导入导出

**.loyi 文件格式**，完整备份与迁移你的记忆数据。

```
memory-export.loyi (ZIP)
├── manifest.json          # 导出元信息
├── memories/            # 每条记忆的完整数据
├── graph/               # 节点与关系数据
└── attachments/         # 图片/音频原始文件
```

- **导出** — 一键导出全部记忆，图谱，附件
- **导入** — 智能去重，已存在的记忆自动跳过

## 技术架构

```
MemoryWeaver/
├── app.py                          # Flask 主应用 + SocketIO 实时通信
│
├── services/                       # 业务逻辑层
│   ├── llm_service.py             # LLM 接口封装（统一调度多提供商）
│   ├── memory_service.py           # 记忆 CRUD 与检索
│   ├── graph_service.py            # NetworkX 知识图谱构建与管理
│   ├── embedding_service.py         # FAISS 向量索引与相似度搜索
│   ├── temporal_extractor.py        # 多模态时间信息解析
│   ├── enhanced_knowledge_extractor.py  # 深度知识抽取
│   ├── prediction_service.py        # 图谱智能预测引擎
│   └── export_service.py           # .loyi 导入导出服务
│
├── templates/                       # Jinja2 HTML 模板
│   ├── index.html                  # 单页应用主入口
│   └── components/                  # 可复用组件（header / sidebar）
│
├── static/                          # 静态资源
│   ├── js/                          # D3.js 图谱 / Socket.IO 通信
│   ├── css/                         # 模块化样式（12 个 CSS 文件）
│   └── uploads/                    # 用户上传文件存储
│
└── data/                            # 本地持久化存储
    ├── memories.json                # 记忆数据
    ├── graph.json                   # 图谱数据
    ├── faiss_index.bin              # 向量索引
    └── relation_types.json          # 关系类型配置
```

| 层级 | 技术选型 |
|:-----|:---------|
| 后端 | Flask 3.0+ · Flask-SocketIO · NetworkX · FAISS |
| 前端 | D3.js v7 · Socket.IO · html2canvas |
| AI | MiniMax / OpenAI / Qwen · Sentence Embeddings |
| 存储 | JSON 文件 + FAISS 向量索引 |
